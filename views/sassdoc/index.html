    respond-to

    @mixin respond-to($breakpoint) { ... }

    Description

    Breakpoint manager Prints out a @media block based on the given value. If the value is a map, it will be printed as is. If the value is a string, it will be used as a key from the $breakpoints map.

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $breakpoint

    Breakpoint or breakpoint name

    		Map or String—none

    Content

    This mixin allows extra content to be passed (through the @content directive).

    Example

    .foo {
  color: red;

  @include respond-to('small') {
    color: blue;
  }

  @include respond-to((min-height: 500px)) {
    height: 550px;
  }
}

    Throws

    • Invalid query #{inspect($breakpoint)} for @media.

    Requires

    • [variable] breakpoints
